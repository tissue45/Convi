# 운영 체크리스트

배포 전/후 점검용 간단 체크리스트입니다.

## 배포 전

- [ ] `npm run lint` 통과
- [ ] `npm run build` 성공
- [ ] `.env` 값 누락 없음
- [ ] 결제/지오코딩 키 정상 주입
- [ ] 보호 라우트 접근 정책 확인

## 배포 후

- [ ] 랜딩(`/`) 진입 확인
- [ ] 인증(`/auth`) 진입 확인
- [ ] 고객/점주/본사 핵심 페이지 접근 확인
- [ ] 결제 성공/실패 라우트 동작 확인
- [ ] `/health` 응답 확인

## 장애 대응 기본 절차

1. 콘솔/네트워크 에러 수집
2. Supabase 연결/권한 상태 확인
3. 환경변수와 배포 설정 재검토
4. 최근 변경 파일 범위 축소 확인

## 운영 주의사항

- 민감 키는 코드에 하드코딩하지 말고 환경변수로 관리
- 역할 권한 관련 오류는 라우트 가드와 백엔드 정책을 함께 확인
- 배포 전후 동일 시나리오로 회귀 테스트 수행
